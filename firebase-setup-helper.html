Step 1: Create Firebase Project

Click the button below to open Firebase Console in a new tab:

  1. Click "Add project" or "Create a project"
  2. Project name: moe-superbowl-squares
  3. Disable Google Analytics (not needed)
  4. Click "Create Project"

Step 2: Add Web App

  1. In your new project, click the web icon (</>)
  2. App nickname: MOE Squares
  3. Don't enable Firebase Hosting
  4. Click "Register app"
  5. Copy the firebaseConfig object that appears
  6. Paste it in the box below:
Configuration saved! Proceed to Step 3.

Step 3: Enable Realtime Database

  1. In Firebase Console sidebar, click "Realtime Database"
  2. Click "Create Database"
  3. Choose location: United States (us-central1)
  4. Start in test mode
  5. Click "Enable"

Step 4: Configure Database Rules

  1. In Realtime Database, go to the Rules tab
  2. Replace existing rules with this:
{
  "rules": {
    "messages": {
      ".read": true,
      ".write": true,
      "$messageId": {
        ".validate": "newData.hasChildren(['username', 'text', 'timestamp']) && newData.child('text').val().length <= 200 && newData.child('username').val().length <= 20"
      }
    }
  }
}

Then click "Publish" in Firebase Console

Step 5: Deploy Updated Config

After saving the config in Step 2, your updated firebase-config.js file is ready.

Deploy to Vercel:

vercel --prod --yes

Or push to GitHub (Vercel will auto-deploy):

git add .
git commit -m "Add Firebase configuration"
git push origin master
